Frequently Asked Questions

About these image-only sample PDF files.

What images are inside each PDF?
Each PDF contains 2 pages, each filled edge-to-edge with a different Canvas-generated PNG image. Page 1 has a full-page landscape scene — gradient sky, mountains, sun, and geometric shapes. Page 2 has a full-page abstract composition — radial patterns, overlapping circles, geometric art, and colorful shapes. All images are unique to each file size's color palette. There is no text in the PDF at all.
How are the images generated?
Images are drawn on an HTML Canvas element in your browser using gradients, arcs, rectangles, and paths. They are exported as PNG and embedded into the PDF using pdf-lib's image API. No external images or servers are involved — everything is generated locally.
Are these valid, openable PDF files?
Yes. Every file is a fully valid PDF 1.7 file that opens correctly in Adobe Acrobat, Chrome, Firefox, Edge, Preview on macOS, and all other standard viewers. The embedded PNG images display at full page size.
Why do large files take longer to generate?
After embedding the images, additional padding bytes are generated in JavaScript to reach the exact target file size. For 50MB and 100MB files this takes 10–25 seconds. The progress bar shows real-time status.
Are these files safe to download?
Yes. All files are generated locally in your browser. Nothing is downloaded from an external server and no data leaves your device. The PDFs contain only embedded PNG images and padding data — no scripts, no macros, no executable code.
