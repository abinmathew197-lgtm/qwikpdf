Frequently Asked Questions

Everything about these black & white text sample PDF files.

What content is inside each PDF?
Each PDF contains 2 pages of pure black and white text. Page 1 is a structured cover page with a title, file information table, and a description section. Page 2 is a content page with multiple titled sections and Lorem Ipsum paragraph text. There are no images, no colour fills — just clean text on a white background.
How are the PDF files generated at an exact size?
Each PDF is generated in your browser using the pdf-lib JavaScript library. After building the text content, the remaining bytes needed to reach the exact target size are appended as a raw data stream — a standard, valid PDF technique that does not affect how the file opens or renders.
Are these valid, openable PDF files?
Yes. Every file is a fully valid PDF that opens correctly in Adobe Acrobat, Chrome, Firefox, Edge, Preview on macOS, and all other standard PDF viewers. The files contain real readable text and conform to the PDF specification.
Why does the 50MB or 100MB file take longer?
Large files require generating significant padding data in your browser's JavaScript engine. A progress bar shows real-time generation status. Smaller files (under 10MB) generate instantly.
Are these files safe to download?
Yes. Everything is generated in your browser — nothing is downloaded from an external server and no data leaves your device. The PDFs contain only text and padding data. No scripts, no macros, no executable code.
